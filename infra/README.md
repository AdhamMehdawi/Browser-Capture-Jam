# VeloCap — Azure infrastructure (Terraform)

All infrastructure-as-code for VeloCap on Azure lives here.

## Target

- **Subscription:** The JUMP App Subscription (`fb42fb29-4fcc-40a9-9ff0-21ef191674af`), Sponsorship
- **Tenant:** `malakalsaffarthejumpapp.onmicrosoft.com` (`0451fcf1-2dd9-4eff-90e3-8396b9df62d9`)
- **Region:** `uaenorth`
- **Resource groups:** `VeloCap` (workload), `velocap-tfstate-rg` (state backend)

## Layout

```
infra/
├── bootstrap/            One-shot: creates the Storage Account that holds
│                         Terraform remote state. Uses LOCAL state.
├── modules/              Reusable building blocks, one per resource type.
│   ├── log-analytics/
│   ├── key-vault/
│   ├── postgres-flex/
│   └── ... (acr, app-insights, storage-account, container-apps-env,
│            container-app, static-web-app — to be added)
└── envs/
    └── dev/              Composes modules for the dev environment.
        (prod/ follows identically with different tfvars)
```

## Run order (first time)

```bash
# 0. Create the two top-level resource groups manually (one-time)
az group create -n VeloCap -l uaenorth
az group create -n velocap-tfstate-rg -l uaenorth

# 1. Bootstrap the state backend (local state, one-shot)
cd infra/bootstrap
terraform init
terraform apply

# 2. Copy the backend_config_snippet output into envs/dev/backend.tf

# 3. Apply dev env against remote state
cd ../envs/dev
terraform init
terraform apply
```

## Conventions

- **Naming:** lowercase `velocap` + service suffix + env suffix, e.g. `velocap-kv-dev`, `velocap-pg-prod`, `velocapstprodaue01`.
- **No sub-scope role assignments.** Deployer is Contributor-only. Key Vault uses access policies (not RBAC), Storage and ACR use admin keys stored in Key Vault. This is an explicit trade-off to keep Tareq's role minimal — see the project plan.
- **All secrets in Key Vault.** The Postgres admin password is generated by `random_password` inside the env, written to Key Vault, and never exposed in tfvars or state in plaintext form.
- **Minimal SKUs everywhere.** Burstable Postgres, ACA min=0 for dev, SWA Free, ACR Basic. Upsize later.
